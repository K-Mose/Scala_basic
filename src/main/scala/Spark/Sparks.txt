SPARK
인메모리 기반의 대용량 고속 처리 엔진으로 범용 분산 클러스터 컴퓨팅 프레임워크
  speed
    인메모리 기반의 빠른 처리
  Ease of Use
    다양한 언어 지원 (Java, Scala, Python, R, SQL)을 통한 사용의 편의성
  Generality
    SQL, Streaming, 머신러닝, 그래프 연산 등 다양한 컴포넌트 제공
  Run EveryWhere
    Yarn, Mesos, Kubernetes 등 다양한 클러스터에서 동작 가능
    HDFS, Casandra, HBase 등 다양한 파일 포맷 지원

인메모리 기반의 빠른 처리
  - 인메모리 기반의 처리로 맵리듀스 작업처리에 비해 디스크는 10배, 메모리 작업은 100배 빠른 속도
  - 맵리듀스 작업의 중간 결과를 디스크에 쓰기 때문에 IO로 인하여 작업 속도에 제약이 생김
  - 스파크는 메모리에 중간 결과를 메모리에 저장하여 반복 작업의 처리 효율이 높음
    ※ 디스크 vs 메모리

다양한 컴포넌트 제공
  - 스파크는 단일 시스템 내에서
    Spark Streaming -> Stream 처리
    Spark SQL -> SQL 처리
    MLib -> MachineLearning 처리
    GraphX -> Graph Processing 지원
  - 추가적인 소프트웨어의 설치 없이도 다양한 애플리케이션 구현 가능
  - 각 컴포넌트간의 연계를 이용한 애플리케이션의 생성도 쉽게 구현

다양한 언어 지원
  - Java, Scala, Python, R 인터페이스등 다양한 언어 지원
  - 하지만 언어마다 처리하는 속도가 다름, 성능을 위해 Scala 로 개발진행하는 것이 좋음

다양한 클러스터 매니저 지원
  - 클러스터 매니저로 YARN, Mesos, Kubernetes, Standalone 등 다양한 포맷을 지원
  - HDFS,카산드라, HBase, S3 등 다양한 데이터 포맷을 지원하여 여러 시스템에 적용 가능

다양한 파일 포맷 지원 및 Hbase, Hive 등과 연동 가능
  - 스파크는 기본적으로 TXT, Json, ORC, Parquet 등의 파일 포맷을 지원함.
  - S3, HDFS 등의 파일 시스템과 연동도 가능하고 HBase, Hive 와도 간단하게 연동할 수 있음
